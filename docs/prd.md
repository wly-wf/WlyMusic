# Win11本地音乐播放器产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品愿景
一款现代化、轻量化的本地音乐播放器，专为Windows 11设计，兼顾美观与实用性。项目在WSL2 Linux环境中开发，最终打包为Windows桌面应用。

### 1.2 核心目标
- 提供流畅的本地音乐播放体验
- 实现现代化的Win11风格UI
- 保持应用轻量化、低内存占用
- 满足个人使用需求，不追求商业化功能

### 1.3 技术栈
- **前端**: React + TypeScript + Mantine UI
- **桌面框架**: Electron
- **后端/音频处理**: Rust
- **数据存储**: SQLite
- **开发环境**: WSL2 (Linux) + Windows 11

## 2. 目标用户
- 主要用户：开发者本人
- 使用场景：个人电脑上的本地音乐播放与管理
- 用户期望：简洁、高效、美观的播放体验

## 3. 功能需求

### 3.1 音乐库管理
| 功能 | 详细描述 | 优先级 |
|------|----------|--------|
| 音乐导入 | 支持从本地文件夹批量导入音乐文件 | 高 |
| 格式支持 | 支持 MP3、FLAC、AAC、WAV 格式 | 高 |
| 元数据解析 | 自动读取ID3标签等信息（标题、艺术家、专辑、封面、内嵌歌词等） | 高 |
| 音乐库扫描 | 支持增量扫描与全量更新 | 中 |
| 目录设置 | 允许设置一个或多个Windows目录为音乐库，支持手动刷新按钮增量添加新歌曲 | 高 |
| 封面显示 | 显示专辑封面，支持自动从文件读取 | 中 |

### 3.2 播放控制
| 功能 | 详细描述 | 优先级 |
|------|----------|--------|
| 基本播放控制 | 播放/暂停、上一首/下一首、进度条拖拽 | 高 |
| 播放模式 | 顺序播放、随机播放、单曲循环、列表循环 | 高 |
| 音量控制 | 系统音量调节与静音 | 高 |
| 播放队列 | 临时播放队列，可添加/删除歌曲 | 中 |
| 播放速度 | 支持调整播放速度（0.5x-2.0x） | 低 |

### 3.3 歌词显示
| 功能 | 详细描述 | 优先级 |
|------|----------|--------|
| 本地歌词 | 自动读取同目录下的.lrc歌词文件 | 高 |
| 网络歌词 | 支持从网易云/QQ音乐等平台获取歌词（v1.0暂不实现） | 低 |
| 歌词同步 | 滚动显示、高亮当前歌词行 | 高 |
| 歌词搜索 | 手动搜索并下载歌词 | 低 |

### 3.4 歌单管理
| 功能 | 详细描述 | 优先级 |
|------|----------|--------|
| 歌单CRUD | 创建、重命名、删除歌单 | 高 |
| 歌曲管理 | 向歌单添加/删除歌曲，拖拽排序 | 高 |
| 导入导出 | 支持导出为JSON/M3U格式，导入M3U/JSON歌单 | 中 |
| 智能歌单 | 根据歌手、专辑、播放次数等自动创建歌单 | 中 |
| 播放历史 | 记录最近播放的歌曲，可查看播放次数 | 中 |

### 3.5 用户界面
| 功能 | 详细描述 | 优先级 |
|------|----------|--------|
| 主题切换 | 深色/浅色主题，支持跟随系统 | 高 |
| Win11风格 | 亚克力透明效果、圆角设计、流畅动画 | 高 |
| 响应式布局 | 适配不同窗口大小，支持最小化到系统托盘 | 高 |
| 快捷键 | 全局快捷键支持（媒体键、自定义快捷键） | 中 |
| 迷你模式 | 紧凑播放器界面，减少占用空间 | 低 |

### 3.6 系统集成
| 功能 | 详细描述 | 优先级 |
|------|----------|--------|
| 系统媒体控件 | 支持键盘媒体键（播放/暂停、上一首/下一首） | 高 |
| 系统托盘 | 托盘图标、右键菜单控制 | 高 |
| 后台播放 | 关闭主窗口后继续播放（托盘图标控制） | 高 |
| 通知提示 | 歌曲切换时显示系统通知（可选） | 低 |
| 文件关联 | 关联音频文件，双击直接播放 | 中 |

## 4. 非功能需求

### 4.1 性能要求
- **启动时间**: 应用冷启动时间 ≤ 3秒
- **内存占用**: 空闲时 ≤ 100MB，播放时 ≤ 150MB
- **CPU占用**: 播放时CPU占用率 ≤ 5%（标准音质）
- **响应时间**: 用户操作响应时间 ≤ 100ms

### 4.2 可靠性
- **崩溃率**: 月崩溃率 < 0.1%
- **数据安全**: 定期自动备份数据库，支持手动备份/恢复
- **错误处理**: 友好的错误提示，避免数据丢失

### 4.3 兼容性
- **操作系统**: Windows 11 (22H2及以上)
- **音频格式**: 完整支持MP3、FLAC、AAC、WAV
- **分辨率**: 支持1080p、2K、4K显示

### 4.4 可维护性
- **代码质量**: TypeScript严格模式，Rust clippy检查
- **文档**: 关键模块有详细注释，技术架构文档
- **测试**: 单元测试覆盖率 > 70%，核心功能集成测试

### 5.2 模块职责
1. **前端 (React + TypeScript)**
   - UI组件与用户交互
   - 播放控制界面
   - 歌单管理界面
   - 设置页面

2. **Electron主进程**
   - 窗口管理（创建、最小化、关闭）
   - 系统集成（托盘、媒体键、文件关联）
   - Rust IPC桥接（通过Node.js native模块或JSON-RPC）

3. **Rust后端**
   - 音频解码（使用Symphonia库支持多种格式）
   - 元数据解析（ID3、Vorbis注释等）
   - 数据库操作（SQLite CRUD）
    - 文件系统扫描（支持手动刷新与增量扫描，v1.0暂不实现自动监听）

4. **数据存储**
   - SQLite数据库存储：
     - `music_library`: 音乐库元数据
     - `playlists`: 用户歌单
     - `playlist_tracks`: 歌单歌曲关联
     - `settings`: 用户设置
     - `play_history`: 播放历史

### 5.3 关键技术选型说明
- **音频解码**: Rust的Symphonia库，纯Rust实现，支持多种格式，无外部依赖
- **前端播放**: Web Audio API，接收Rust解码后的PCM数据或使用HTML5 Audio播放文件路径
- **IPC通信**: Electron的ipcMain/ipcRenderer + Rust的JSON-RPC over stdio
- **数据库**: SQLite + Diesel ORM (Rust) 或 rusqlite
- **UI组件**: Mantine UI，现代化组件库，支持深色/浅色主题
