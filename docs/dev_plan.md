# Win11本地音乐播放器开发计划 v2

## 1. 项目概述

### 1.1 项目目标
基于PRD要求，开发一款现代化、轻量化的Win11本地音乐播放器，采用Electron + React + Rust技术栈。

### 1.2 核心原则
- 增量开发：每次实现一小部分功能，快速验证
- 技术验证优先：先验证技术栈可行性，再扩展功能
- 用户中心：聚焦核心音乐播放与管理需求

### 2.2 布局分区说明

| 区域 | 宽度 | 功能 |
|------|------|------|
| 顶部栏 | 40px | Logo、导航历史、搜索框、设置/窗口控制 |
| 左侧导航 | 240px | 功能菜单（本地音乐、排行榜等）、我的歌单列表 |
| 主内容区 | 自适应 | 歌单详情、歌曲列表、歌词视图 |
| 播放控制栏 | 64px | 当前播放信息、播放控制、进度条、功能按钮 |

### 2.3 详细设计

#### 顶部栏 (40px)
- **左侧**: Logo "WlyMusic" + 导航后退/前进按钮
- **中间**: 搜索框（圆角，带占位文字）
- **右侧**: 菜单图标、设置⚙️、最小化、最大化、关闭

#### 左侧导航栏 (240px)
- **本地音乐**: 显示全部本地歌曲
- **我的歌单**: 可折叠列表，带+新建按钮
- **播放历史**: 显示最近播放的歌曲
- **播放次数**: 按播放次数排序显示歌曲
- **样式**: 选中项高亮（橙色背景/左侧指示条）

#### 主内容区
- **歌单详情页**: 封面(左侧大图) + 歌单信息 + 播放按钮 + 歌曲列表
- **歌曲列表**: 表格式布局 #、标题、作者、专辑、时长、来源
- **播放详情页**: 全屏歌词视图（点击封面或歌词按钮进入）

#### 底部播放控制栏 (64px)
- **左侧**: 64x64封面 + 歌曲名/艺术家 + 时间
- **中部**: 上一首、播放/暂停(大按钮)、下一首
- **进度条**: 播放栏顶部有一条橙色进度指示线
- **右侧**: 歌词开关、播放模式、音量、播放列表

### 2.4 交互特性
- **进入歌词视图**: 点击底部歌词按钮或点击左下角封面
- **返回**: 歌词页面右上角有收起按钮
- **进度条**: 橙色进度指示线在播放栏顶部
- **主题色**: 橙色(#FF8C42)作为主色调

## 3. 技术架构决策

### 3.1 技术栈确定
| 组件 | 技术选择 | 说明 |
|------|----------|------|
| 前端框架 | React + TypeScript | 现代化UI开发 |
| UI组件库 | Mantine UI | 支持深色/浅色主题 |
| 桌面框架 | Electron | Windows应用打包 |
| 音频处理 | Rust (Symphonia) | 音频解码与播放控制 |
| 数据存储 | SQLite | 本地数据存储 |
| IPC通信 | JSON-RPC over stdio | Rust与Electron通信 |

### 3.2 音频播放架构
**决策**: Rust全权控制音频播放
- Rust负责音频解码、播放、进度控制
- Electron前端通过IPC发送控制命令
- 音频数据通过IPC传输或共享内存
- 优势：最佳性能，统一音频处理逻辑

### 3.3 功能范围
**核心功能 (必须实现)**:
- ✓ 音乐目录设置与手动扫描
- ✓ 音乐库列表展示
- ✓ 基础播放控制（播放/暂停、上一首/下一首）
- ✓ 歌词显示（内嵌歌词 + 本地.lrc文件）
- ✓ 歌单管理（创建、添加歌曲、删除）
- ✓ 设置功能（目录管理、歌词字体大小）

## 6. 风险与应对措施

| 风险 | 影响等级 | 概率 | 应对措施 |
|------|----------|------|----------|
| Rust-IPC通信性能问题 | 高 | 中 | 使用高效序列化，批量传输，考虑共享内存 |
| 音频解码兼容性问题 | 高 | 低 | 使用Symphonia成熟库，提供错误日志 |
| 文件系统扫描性能 | 中 | 中 | 增量扫描，后台线程，进度提示 |
| 数据库并发访问问题 | 中 | 低 | 使用连接池，事务处理，错误重试 |
| 内存占用过高 | 中 | 中 | 懒加载，缓存清理，内存监控 |
